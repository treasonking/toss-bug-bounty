# 샘플 제보서: SSRF 가능성

## 제목
[SSRF] URL 미리보기 기능에서 서버측 외부 요청 수행 가능

## 취약점 유형
- SSRF 가능성

## 대상 URL/API
- Method: `POST`
- Endpoint: `/api/v1/link/preview`
- 앱/웹 버전: `모바일 앱 vX.Y.Z`

## 공격 환경
- 테스트 일시: `2026-04-10 14:05 KST`
- 테스트 계정: A
- User-Agent(credential 고정): 적용
- 프록시/Burp 사용 여부: 사용

## 전제조건
- 로그인 사용자로 URL 입력 기능 접근 가능

## 재현 절차
1. URL 미리보기 요청의 `url` 파라미터에 `http://bugbounty.toss.sb`를 입력합니다.
2. 요청 전송 후 서버 응답과 검증 도메인 측 로그/응답 증적을 확인합니다.
3. 동일 조건에서 일반 비입력 URL 대비 처리 차이를 비교합니다.

## 실제 결과
- 서버가 입력 URL로 요청을 수행한 정황이 확인됨(검증용 도메인 기준)

## 기대 결과
- 서버가 외부 URL 호출이 필요한 기능이라면 강한 검증/허용목록 정책 적용
- 호출이 불필요한 기능이라면 외부 요청 자체가 차단되어야 함

## 파급력
- 서버 내부 네트워크 대상 요청으로 확장될 경우 민감 자원 접근 위험

## 발생 원인 추정
- URL 입력값 검증 미흡 또는 목적지 제한 정책 부재

## 대응방안
- 허용 도메인 기반 allowlist 적용
- 사설대역/메타데이터 주소 차단
- 리다이렉트 추적 제한 및 DNS 재검증 적용
