# 샘플 제보서: 금액변조

## 제목
[Amount Tampering] 결제 생성 요청에서 금액 무결성 검증 미흡

## 취약점 유형
- Amount Tampering (비즈니스 로직 검증 미흡)

## 대상 URL/API
- Method: `POST`
- Endpoint: `/api/v1/payment/create`
- 앱/웹 버전: `모바일 앱 vX.Y.Z`

## 공격 환경
- 테스트 일시: `2026-04-11 11:30 KST`
- 테스트 계정: A
- User-Agent(credential 고정): 적용
- 프록시/Burp 사용 여부: 사용

## 전제조건
- 결제 생성 기능 접근 가능
- 정책 준수: 100원 테스트만 수행

## 재현 절차
1. 정상 요청으로 `amount=100` 결제 생성 요청을 전송합니다.
2. 동일 요청에서 `amount` 또는 `fee` 값을 비정상 값으로 단건 변조합니다.
3. 서버가 변조값을 차단하는지, 또는 서버 재계산을 수행하는지 확인합니다.

## 실제 결과
- 변조 파라미터가 반영된 채 처리되는 정황 확인(예: 검증 오류 미반환)

## 기대 결과
- 서버가 클라이언트 입력 금액을 신뢰하지 않고 주문 원본 기준 재계산/검증해야 함

## 파급력
- 결제 금액 불일치에 따른 금전적 피해 가능성
- 수수료/정산 로직 왜곡 가능성

## 발생 원인 추정
- 서버측 무결성 검증 부재 또는 검증 순서 결함

## 대응방안
- 서버 단에서 최종 금액 재계산
- 클라이언트 전달 금액과 서버 계산값 불일치 시 즉시 거절
- 결제 단계별 서명값/무결성 토큰 검증 강화
