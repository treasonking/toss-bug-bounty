# Findings 운영 가이드

## 목적
- 탐지된 이슈 후보를 제보 전 단계에서 체계적으로 관리
- 중복 제보/증거 누락/재현 실패를 줄이기 위한 내부 기록

## 권장 운영 방식
1. 후보 이슈 발견 즉시 `FINDING_CARD_TEMPLATE_KR.md`로 카드 생성
2. 증거 4종(요청, 응답, 화면, 시간) 채운 뒤 우선순위 분류
3. 재현성 확인 후 `docs/REPORT_TEMPLATE_KR.md`로 승격

## 상태 정의
- `NEW`: 최초 발견
- `TRIAGE`: 재현/영향도 확인 중
- `READY_TO_REPORT`: 제보 가능한 수준
- `REPORTED`: 제출 완료
- `CLOSED`: 오탐/중복/정책 비해당

## 폴더/파일 규칙
- 파일명: `YYYY-MM-DD_<유형>_<기능>_<상태>.md`
- 예시: `2026-04-08_IDOR_profile_TRIAGE.md`

## 주의사항
- 민감정보는 마스킹 후 저장
- 실제 사용자 데이터 포함 금지
- 외부 공유 금지(비밀 유지)
